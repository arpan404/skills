# Redesign

Audit before changing UI.

Audit:

- hierarchy
- layout
- navigation
- density
- typography
- color
- components
- states
- interaction
- accessibility
- responsive behavior
- copy
- product fit

Output:

1. What is broken.
2. Why it hurts UX/product quality.
3. Exact fix.
4. Redesign direction.
5. Implementation plan or code.

Paths:

- Evolutionary: same system, fix quality.
- Pivotal: new visual language, same structure.
- Transformative: new structure and visual language.

Implementation order:

1. Fix information architecture.
2. Fix layout.
3. Fix hierarchy.
4. Fix components.
5. Fix states.
6. Fix interactions.
7. Fix visual system.
8. QA accessibility and responsiveness.
