# Workflow, Collaboration, I18n

## Roles and Permissions

Identify roles early.

For each role define:

- visible data
- allowed actions
- blocked actions
- approval needs
- audit visibility
- escalation path

Permission UX:

- hide impossible actions when they add clutter
- show disabled actions when users need to understand access limits
- explain forbidden states
- avoid dead controls

## Collaboration

Use when multiple users can affect the same object.

Need where relevant:

- presence
- live cursors/selection
- comments/mentions
- assignment
- activity history
- conflict handling
- optimistic updates
- saved/pending/sync failed states

## Localization

Use for global, multilingual, or regional products.

Plan for:

- longer translated strings
- RTL where relevant
- locale date/time/number/currency
- pluralization
- timezone clarity
- legal/regional terms
- no text baked into images
- layouts that survive bilingual labels
