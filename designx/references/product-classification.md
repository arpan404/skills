# Product Classification

Classify before designing. Pick the closest class, then apply its mandatory patterns.

## Marketing

For landing pages, company sites, launch pages, portfolios, events, waitlists, sales pages.

Need: value prop, one primary CTA, visual anchor, proof, feature story, objections/FAQ when useful, final CTA, footer.

Avoid: generic hero, three feature cards, meaningless gradient, no story.

## Editorial

For blogs, magazines, docs, knowledge bases, news, changelogs.

Need: strong type hierarchy, 60-75ch reading width, TOC for long docs, categories/tags, search for docs, related content.

Avoid: wall of text, weak hierarchy, poor mobile reading.

## Commerce

For catalogs, storefronts, marketplaces, subscriptions, B2B ordering.

Need: product imagery, filters/sort, price, availability, variants, quantity, reviews/trust, persistent cart, checkout recovery, guest checkout unless constrained.

Avoid: hidden costs, early account wall, unclear CTA.

## SaaS App

For productivity, CRM, HR, finance, devtools, project management, collaboration.

Need: persistent nav, search/command for complexity, settings/account, notifications, onboarding/empty states, keyboard support for power tasks.

Avoid: marketing-page style inside operational software.

## Dashboard / Analytics

For executive, operational, analytical, personal, IoT, financial dashboards.

Need: dashboard type, primary metric/alert, visible date/filter controls, correct chart types, drill-down, raw table/list.

Avoid: 20 equal cards, decorative charts, no decision path.

## Admin / Internal Tool

For CMS, user management, access control, moderation, fulfillment, config, support ops.

Need: sortable/filterable tables, bulk actions, advanced filters, audit log, confirmation for destructive actions, import/export, role-aware permissions.

Avoid: consumer spacing, no bulk operations, no audit trail.

## Mobile App

For consumer utility, social, service, commerce, productivity, field worker apps.

Need: native-feeling nav, safe areas, 44pt/48dp targets, gestures, bottom/stack navigation, keyboard-aware forms.

Avoid: desktop UI squeezed into phone width.

## Cross-Platform

For React Native, PWA, Electron/Tauri, shared web/mobile systems.

Need: shared tokens, platform-specific nav/gestures, adaptive layout, offline/install behavior for PWA, keyboard/right-click for desktop.

Avoid: one UI that feels wrong everywhere.

## AI / Agent Workspace

For AI chat, agent IDEs, assistant workspaces, prompt tools, research/coding assistants.

Need: conversation, artifact/editor panel, tool timeline, sources/citations where relevant, approval queue, memory/context panel, attachment composer, streaming/retry/stop/regenerate/branch.

Avoid: simple chat box with no traceability or control.

## Developer Environment

For code editors, API clients, database GUIs, CI/CD, logs, local devtools.

Need: resource tree, tabs, editor/workspace, terminal/log panel, problems panel, command palette, status bar, shortcuts, context menus.

Avoid: generic SaaS dashboard.

## Creative / Canvas Tool

For design tools, whiteboards, diagram/video/audio/3D/map editors.

Need: canvas, toolbar, layers/object list, inspector, zoom/pan, selection, snapping/alignment, precision input, undo/redo.

Avoid: page-based layout for spatial work.

## Operations / Field Management

For construction, logistics, manufacturing, maintenance, facilities, scheduling.

Need: schedule/map/crew/equipment views, task states, field notes/photos, offline/retry, handoff, escalation, audit trail.

Avoid: executive dashboard with no operational actions.

## Finance / Accounting / Trading

For ledgers, invoices, budgets, reconciliation, trading, portfolio tools.

Need: precise numbers, tabular figures, audit trails, permissions, approvals, exports, reconciliation states, warning/error distinction.

Avoid: decorative charts without transaction detail.

## Healthcare / Clinical

For EHR, scheduling, patient intake, care coordination, clinical ops.

Need: patient identity clarity, critical info hierarchy, status warnings, audit history, privacy states, conservative UI.

Avoid: vague colors, hidden critical info, consumer decoration.

## Education / LMS

For courses, learning apps, assessments, tutoring, school admin.

Need: progress, lesson flow, assignments, feedback, cohorts/classes, accessibility, parent/teacher/student roles where relevant.

Avoid: static content pages with no learning flow.

## File / Document Manager

For drives, DAM, contract/doc workflows, media libraries.

Need: list/grid toggle, preview pane, metadata, search, filters, bulk actions, versions, permissions, activity, upload progress.

Avoid: simple list with no preview or organization model.
